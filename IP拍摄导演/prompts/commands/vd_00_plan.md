# /vd_00_plan - 拍摄计划（先计划，不写成稿）

读取 **项目根/outputs/inputs/brief.md**（必要时读取 **项目根/outputs/inputs/ip_profile.md** 与 **项目根/outputs/inputs/refs.md**）。
**先确定/创建本话题工作目录**：时间戳=当日 YYYYMMDD，主题概括=brief「话题一句话」前 20 字（去掉路径非法字符）；在 **项目根/outputs/** 下创建 **`{时间戳}-{主题概括}/`**（若不存在），将 **项目根/outputs/inputs/brief.md** 备份到该目录 **00_brief.md**。
生成 **项目根/outputs/{时间戳}-{主题概括}/00_plan.md**（计划归属本话题工作目录）。**禁止在 skill 目录下写入；所有文件在 项目根/outputs/ 下。**

**场域校准必须先于计划**：产出的 00_plan.md 第一段必须是「0) 场域校准」，再写 1)～6)。没有共同探究状态，后面再完美的结构也只能剪出「像在说话，但没聊透」的内容。

## 必须包含

**0) 场域校准（必须输出，放在计划最前）**

- **今日共同目标（1 句）**：把____聊透，而不是完成拍摄
- **打开深度**：浅 / 中 / 深（知识类重点是「愿意顺着问题走到尽头」）
- **三个状态检查**（每项 1 句话自检）：
  - 探究欲：
  - 清晰表达：
  - 放下功利：
- **若状态不对**：本次策略（三选一或组合）——缩短目标 / 改为单点问题 / 先热身 3 问

1) **一句话结论**（可转述，燃模式 ≤16 字更好）
2) **隐藏结构**（钩子-冲突-关键变量-抓手-动作-人味收尾）
3) **5 层提问梯子路线图**（每层目标 + 关键问题）
4) **必须拿到的 3 类可剪片段**（HOOK / MECH / ACTION / HUMAN）
5) **风险点与救场策略**（3 条）
6) **6 发动机自检**（勾选）

## 文件契约

- 输入：**项目根/outputs/inputs/brief.md**（及 ip_profile.md、refs.md 若存在）
- 输出：**项目根/outputs/{时间戳}-{主题概括}/00_plan.md**（覆盖）；同时在该目录创建/覆盖 **00_brief.md**（brief 备份）。不写入 plans/ 或 current_run_dir。

## 执行

先执行本命令，确认计划后再执行 vd_01～vd_04 生成现场用文件。
